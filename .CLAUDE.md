# Last updated: 2026-03-11 | Evidence: docker-compose.yml, Makefile, docker/*/app.py

## What It Is
Local-first AI chat platform. Multi-container Docker Compose stack. No cloud dependency.
LLM served by LM Studio at `host.docker.internal:1234`. MCP server (96 tools) on `:8096`.
Pinned model: `liquid/lfm2-24b-a2b` (set via `IMAGE_GEN_MODEL` in `.env`).


## aichat-mcp SSE Protocol (docker/mcp/app.py)
```
GET  /sse           Client connects; receives endpoint event → /messages?sessionId=<id>
POST /messages      Client sends JSON-RPC requests
GET  /health        Health probe
GET  /tools         List all tools (used by generate-lmstudio-json)
```
Screenshot support: human_browser container writes to /workspace, bind-mounted read-only
at /browser-workspace inside aichat-mcp.

## Critical Rules
- `aichat-data` consolidates 5 former services. Do NOT split them back.
- Port assignments are fixed — downstream MCP clients depend on them.
- `POSTGRES_PASSWORD` must come from `.env`; use `${POSTGRES_PASSWORD:?...}` pattern.
- `entrypoint/` scripts are fragile — wrong changes cause silent container boot failures.
- `docker/data/` schema changes require a migration file, not in-place edits.
- whatsapp container is Node.js, not Python — different dependency management.
- `aichat-data` and `aichat-mcp` have an `extra_hosts` entry for `host.docker.internal`.
